Writing your first shader

December 21, 2017, Robin Opheij

In this tutorial I will go over how to setup a basic shader whilst explaining everything in the progress.

prerequisite

Most unity developers know that unity works with a component system, so does the rendering system. Unity had multiple rendering components depending on what type of object you want to render. in our example we are using the meshrenderer.

Every object only uses one renderer but a renderer can have as many materials that it needs and every material can have its own shader.

Creating the shader file

I think it's good practice to have a shader folder for all my shaders so create one first. (I'm going to assume that you know how to make a folder).

Once you made the shader folder, open it and go to the create menu.

Now hover over shader and you'll see:

which one do I choose?" you might ask. and for now that doesn't really matter since we are going to delete it all the code in the shader so I can explain it from the ground up. it's like a unlit shader so you might aswell create that. After you have created the file you can open it with your prefered IDE, I use visual studio but it doesn't really matter.

The shader

Now that you have created and opened the shader you can go ahead and delete all the code in it, So we can rewrite it from scratch. First we give it the outer layer, with the (sub)folder and name

between the "" we say Custom which is under what subfolder the shader can be found in the materials shader dropdown menu, and the name will be whatever you name the part after the last /. this does mean that you can have as many subfolders as you want. Let's make the properties tab, The properties tab is basicly the place where we define our "public" variables, we see these in our materials inspector and the names of these variables are what we call if we want to access them in a script.

As you can see here inside the properties tab we defined a few things. lets go over them; in this example _Color is the variable name ("Color") is how it will be shown in the inspector and (, Color) is the value type. We then defined the default for the shader which in this case is (1,1,1,1) After we defined our properties we move on to create the subshaders and passes. You can have multiple subshaders and passes but for now we'll just create one.

Everything outside of the pass {} is part of the shaderlab which sends the cg data from withing the pass{} over to unity's rendering system. Inside the pass we are first going to tell the shader how the vertex and fragment functions are going to be called.

One of the predefined structs in UnityCG.cginc.

float4 = type of variable, vertex = variable name, POSITION = data needs to be pulled. The next struct we make is the vertex to fragment struct.
