How to write your fist Neural Network

November 30, 2018, Robin Opheij

In this blog we will go over the easiest part of machine learning.

The hello world of machine learning.

Right now we are going to write a programm that when we give it a bunch of trainingdata and some new inputs it will give us the correct results.


Since I write most of my projects in c# and since c# is a wildly used program I will write this tutorial for c#. It should be pretty easy to follow even if you are used to different languages. If you still prefer to write this in your main language then please do, I would love to see all the different Implementations of this tutorial. On the git for this tutorial I might even go ahead and add different languages in different branches.


So what it the "hello world" to neural networks. well it's a nn that can compute a XOR function, this idea has it's roots in binary logic and stands for exclusive-or. So basically it finds out if a if statement in binary should return true or false. So if the result of XOR is 1 then one of the inputs is 1(true) and the other is 0(false). If t.hey are both false or true then the result will be 0

So now lets get into the hierarchy of the neural network we are going to create.

So now that you know what results we are after I'm going to explain how we are going to do this. We are going be be doing this trough a backpropagation algorithm. This works by first generating results by having random inputs and random outputs. then checking those results trough the expected results and then adjust the weights and biases accordingly and restart that whole process. this process is what we call supervised-learning.

So first we "forward propagate" which does that first bit of calculating the output values and then the "back propagation" will adjust the weigths and biases accordingly.


So how and what are we going to do:

  • 1. Initialize all neurons with random values between 0 - 1.
  • 2. Calculate the output trought the feedforward
  • 3. Calculate the global average error (GAE).
  • 4. Adjust the neurons with this GAE.
  • 5. Calculate Hidden layer neurons error (GAE split). Then adjust the neurons accordingly.
  • 6. repeat this process except for step 1 until the GAE gets minimal.

So every neural network need a "activation function" this is also called a transfer function sometimes. this could be a sigmoid or sine function. However we are going to use a sigmoid function. we adjust this function until it gives us the correct outputs.

So now let's be honest neural networks are actually just tons of math. complex math at that if you don't know what you are looking at. I know that some people aren't like huge math heads so I'll keep this short.
so I said something about derivative function and normal function which in math looks like f(x) <-- normal, f'(x) <-- derivative. basically f(x) means function of x which is y, but f'(x) means deltax/deltay. So let's start with calculating the global error. We do this by:
globalOutput = f'(output) * (excpectedOutput - actualOutput) as you see we see a few variables here for instance actualOutput. Now how do we calculate that. This isn't too hard so let me quickly show you:
actualOutput = f(weights[0] * inputs[0] + weights[1] * inputs[1] + bias).

to adjust the output neuron you just do weight += globalError * inputs and bias is the same except you dont multiply it by the input.
To calculate the error of each neuron you just use this formula:
error1 = f'(x) * globalError * weight
next thing to do is adjusting the hidden layers which is just as easy by just doing
weight += error * input; and the bias by
bias += error.
